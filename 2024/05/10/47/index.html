群晖 NAS 指南（一）系统篇

1 硬件

1.1 主机

  • Hasee Mini PC2 D1 （闲鱼148￥=主机无后盖198￥-USB2.0损坏50￥）

    CPU：Intel 赛扬 四核四线程 J3160（Braswell）
    主板：AMI（安迈） Aptio CRB（英特尔 PCI 标准主机 CPU 桥）
    核显：Intel GMA HD 400 （320-640 (Boost) MHz）
    内存：Kingston（英睿达） DDR3L 1600Hz 4 GB
    硬盘：Phison（群联）PS3111-S11 mSATA TLC 128 GB
    无线网卡：RTL8188EE 150Mbps（支持蓝牙4.0）
    有线网卡：1000Mbps 以太网卡
    数据接口：3×USB2.0（坏） + 2×USB3.0
    音频接口：耳机 / 麦克风两用接口
    视频接口：HDMI（不可用）、VGA
    网络接口：RJ45（一个）
    读卡器：多合一读卡器
    机箱尺寸：153.4×153.4×38 mm

1.2 后续改装

硬盘：WD5000LPCX 5400rpm 500GB 20￥ + ST2000LM003 5400rpm 2TB 281￥ + 2.5寸硬盘盒 11￥
外壳：外壳一套 30￥
线材：飞利浦 HDMI2.0 9.5￥ + sata3延长线 7.65￥ + HDMI 转 VGA 9.85￥
电源：华硕 ADP-65DW B（19V 3.42A）28.88￥
系统引导：爱国者 USB2.0 32GB U盘 23.78￥
来电自启：智能计量插座 淘宝临时工 17.94￥

2. 系统

2.1 系统引导

2.1.1 引导盘制作

  1. 下载 ARPL 原版引导工具
  2. 将下载好的压缩包解压，得到一个后缀为img的文件
  3. 使用写盘工具Rufus将镜像文件写到u盘中，点击选择,找到解压好后缀为img的文件，其他保持默认，点击开始。（注意设备选择引导 U 盘）
  4. 当状态栏提示准备就绪表示文件写入成功，点击关闭按钮，拔出引导U盘即可。

2.1.2 引导编译

  1. 将 NAS 主机接好鼠标键盘显示器电源（HDMI 无法连接显示器，只能使用 VGA 接口），然后将引导 U 盘插入主机的后置接口，打开电源不断按 DELETE（或者 F2） 进入 BIOS，在 boot 选项中将u盘设置为第一启动项。完成后保存并重启机器即可。

  2. F10保存机器将重启，进入如下页面，选择第一个。

  3. 系统会自动跑代码，然后进入到如下页面。

  4. 同一局域网电脑，打开浏览器输入上面步骤中标红的ip地址和端口，教程中为 192.168.2.193:7681，首先将语言设置为中文方便后续操作，使用鼠标点击或者键盘挪动光标到 Choose a language，按回车按键然后选择 zh_CN 切换到中文。

  5. 选择型号➡选择918+，然后确认。

  6. 选择版本➡选择DSM7.1，然后确认。

  7. 弹出的版本页面，点击URL会自动下载当前引导的系统文件（建议使用此方式下载，避免在群晖官网下载错系统），下载好的系统文件后缀为pat，然后点击确认。

  8. 选择编译引导。

  9. RR会自动完成引导的编译。

  10. 编译好引导后会自动跳转到主菜单的启动选项，直接点击启动即可。

  11. 过1-2分钟后使用页面弹出的IP地址访问即可进入到系统安装步骤。

2.2 系统安装

  1. 浏览器输入192.168.2.193:5000进入系统安装页面，点击安装。
  2. 选择从计算机手动上传.pat文件，文件为引导编译步骤7中下载的文件，然后点击下一步。
  3. 勾选我了解这些硬盘上的所有数据将被删除，然后点击继续。
  4. 输入产品型号点击删除。
  5. 系统安装中，大概需要5-10分钟左右。
  6. 页面倒计时结束后刷新页面即可进入初始化步骤。
  7. 安装完成之后进入初始化页面。

系统安装完毕后，U盘必须一直插在群晖主机上，否则无法开机

2.3 系统初始化

  1. 设置用户名和密码，然后点击下一步。
  2. 更新选项选择第三个，黑群晖所有更新都建议手动安装。
  3. 经过简单的设置，成功进入群晖系统。

    存储设置中，系统盘一般选择 brtf，存储盘选择 ext4

  4. 系统安装成功。
  5. 可根据我不是矿神的教程进行系统优化

2.4 系统更新

黑群晖安装完成后，不太建议升级更新系统版本，但是由于系统 7.1 版本中 docker 容器的问题，让我不得不想着去升级更新到7.2的版本，DSM 7.2 更新最引人注目的改变是 Docker 套件的变化，现已正式更名为 “Container Manager”(容器管家)。Container Manager 在DSM 7.2 中支持了 Docker Compose，这一功能大幅增加了 Docker 的可玩性和便利性，使用户能够通过配置文件直接启动容器。Container Manager 在注册表中新增了阿里云与 Dcoker Hub（v1）两个仓库，解决了原 Docker Hub 被隔离的窘境。版本更新的更多功能可以到群晖官网去了解！

  • 更新系统前建议拔出系统盘外的其他存储盘，并做好备份

APRL 引导配置文件存放在U盘里，需要修改相关配置时可以拔下U盘更新配置文件

  1. 拔出群晖主机上的 U 盘并插入电脑上，复制 grub.cfg 文件至电脑：打开 DiskGenius ，在左侧进入U盘中 boot->grub，选择grub.cfg 右击，选择“文件复制到“桌面”。

  2. 复制 user-config.yml 文件至电脑：打开 DiskGenius ，在左侧进入U盘中 RR1(0)->user-config.yml，选择grub.cfg 右击，选择“文件复制到“桌面”。

3.修改 user-config.yml 参数：用记事本打开桌面上的 “user-config.yml” 文件，这里需要修改三类参数，参数修改完成，点击保存即可。

3.1 第一类：U盘对应的PID和VID

3.2 第二类：硬盘接口对应的 SataPortMap 和 DiskIdxMap 参数修改，SataPortMap 和 DiskIdxMap 参数获取方式见：https://guoqing.blog.csdn.net/article/details/129287254

3.3 第三类：洗白所需修改的 sn 和 mac，如果无需洗白，这里则无需修改。在 sn 和 mac 处输入购买的洗白码对应数值即可。

  1. 更新U盘中的 grub.cfg 文件：打开 DiskGenius，进入 boot->grub ，将更新后的 grub.cfg 文件拖入进去，会提示确定将文件复制到目录，点击 “是(Y)”。其后，提示文件已存在，是否替换已存在文件，点击替换即可。随后点击完成，拔下U盘插入群晖主机中。

注：***TANK电玩 公开群晖半洗白码***

2.5 来电自启

如果想使用来电自启，请按照以下步骤进行配置

  1. 开机时狂按 ”DELETE“ 或者 ”F2“ 键进入 BIOS 设置

  2. 选择 Chipset -> South Bridge，按回车。

  3. 更改 Restore AC Power Loss 选项为 ”Power On“

  1. 点 F10 保存设置，退到主界面后再点 F10（或者直接选择 File->Save Changes and Exit）,提示是否保存并退出，默认为 YES，点回车即可退出，自动重启。

3. 应用设置

添加第三方套件源：
套件中心->设置->套件来源->新增
（我不是矿神源：https://spk7.imnks.com/）

通过 Cloud Sync,您可在 Synology NAS 与各种共有云(如 Dropbox、阿里云 0SS 和 Google 云端硬盘) 之间无缝进行同步、分享文件。

可添加 Onedrive 商业版，调用 API 提高 E5 订阅存活率

3.2 Active Backup for Microsoft 365

Active Backup for Microsoft 365 提供了一个集中式界面，其自动让您能够高效备份和管理数据，连续备份模式可最大限度降低数据丢失风险，Active Backup for Microsoft 365 Portal 支持轻松还原，等等。

可添加 Onedrive 商业版，调用 API 提高 E5 订阅存活率

3.3 Synology Photos

Synology Photos 提供一系列灵活的智能功能，让摄影爱好者和家庭用户都可以轻松管理照片

需要搭配Advanced Media Extensions (AME) 来建立索引及显示缩略图

3.4 Advanced Media Extensions (AME)

Advanced Media Extensions (AME) 可让您在 Synology NAS 上查看高效率现频编码 (HEVC) 文件。
安装 AME 后，登录 Synology 账户（正版专属）以安装解码包

能够成功激活的前提是你的群晖为半白（有正确的SN号）

  • 脚本激活
  1. 将以下代码保存为 .py 文件。例如：ame.py
  2. 将 ame.py 保存到群晖中。查看保存文件的属性，记住文件的位置，例如：/volume2/photo/ame.py
  3. 打开套件中心安装 Advanced Media Extensions 套件。安装完成后打开需要你登录，这里直接关闭不用管。
  4. 在控制面板的任务计划中，新建一个计划任务使用用户自定义脚本，用户账户使用root。
  5. 任务设置中的用户自定义脚本输入：python 后面跟刚才我们保存的文件位置。
  6. 保存任务后在任务上点击右键运行，运行完成后稍等十几秒，打开 Advanced Media Extensions，我们看到解码包已经开始安装了。

  7. 最后进入Synology Photos的设置，重建HEIC文件索引即可。

    如果没有开始安装请彻底删除 Advanced Media Extensions 后重装并再次手动运行我们的脚本。

3.5 VPN Server

VPN Server 提供了简单易用的 VPN 解决方案，可让 Synology 产品成为 VPN 服务器，供客户端以安全的方式从远程位置连接至您的私人局域网，并支持 PPTP、OpenVPN 以及 L2TP/IPSeC 等服务

由于政策所限，需要到非中国网站下载群晖VPN Server，这里建议到群晖台湾网站下载（繁体相比其他语言来说更易理解）

  1. 进入群晖香港网站，根据型号、系统版本下载对应的套件

  2. 下载 VPN Server

  3. 下载完成后，打开套件中心——手动安装——选择下载的套件

  4. 安装完成后打开 “VPN Server” —— 权限 这里可以根据需求设置账户使用 VPN 权限

  5. 在设置 VPN Server 区域中可以设置三种类型 VPN，只需要在使用的类型中勾选 “启用VPN服务器” 即可开启，最大联机数量默认为 5，可以调，机型不同，最大联机数量也有区别

  6. 设置 IKE 验证，预共享密钥，设置好后点完成

  7. 设置完成后需要在路由器中映射对应的 VPN 端口，设置方法与 Web 映射一致。

    端口映射好后，就可以通过Win10电脑自带的 VPN 进行连接

    1. Win10电脑打开网络设置——VPN——添加 VPN 连接
    2. 输入相关信息，点击保存（服务器名称输入域名无效，只能是 IP 地址）
    3. 设置完成后点击——连接
    4. 连接成功后可以使用内网IP进行网络驱动器映射

    上述设置无法远程连接暂未解决此问题，建议使用OpenVPN

  8. 在L2TP/IPSec协议中，我已经说明了防火墙和路由器里的端口映射，这里需要添加OpenVPN协议采用默认的端口：1194，路由器端口映射，才可以外网访问！

  9. 你只需要点击启用，一切采用默认配置，端口可以修改，建议采用默认的1194

  10. 先点击应用，再导出配置文件

  11. 解压配置文件，使用文本编辑器打开VPNConfig.ovpn文件，将第四行中的”YOUR_SERVER_IP”替换成你nas的外网IP或域名，保存关闭。

  12. 下载客户端 OpenVPN GUI，启动OpenVPN GUI客户端，输入 NAS 的账号密码：

注意：

  • Openvpn连接的时候提示：No server certificate verification method has been enabled 错误；你需在安装目录 \OpenVPN\config 下的 OpenVPN Config File（.ovpn）里面最后一行加上 remote-cert-tls server
  • 勾选启动 IPv6 服务器模式来启动 OpenVPN 服务器，以传送 IPv6 地址。您必须先在控制面板 > 网络 > 网络接口中，透过 6in4/6to4/DHCP-PD 取得 Prefix，并在此页面中选择该 Prefix。

VPN文件传输速度比WebDav慢，只是安全性更高

3.6 Docker

Docker 是一个轻量级虚拟化应用程序，它可让您运行世界各地的开发人员在 DSM 上创建的数以千计的容器，Docker Hub是使用广泛的内置映像存储库，可让您从其他优秀开发人员那里找到共享的应用程序

自从2023年5月中旬，著名Docker 容器平台： https://hub.docker.com “不知” 何种原因国内均无法正常访问了。

3.6.1 Docker 访问

Docker 搜索应该是只能使用官方地址 registry.hub.docker.com，被墙导致：查询注册表失败

注意：被墙无法通过修改 hosts 来解决，只能魔法。

解决办法：

  • 通过 SSH Docker Pull 拉取镜像

    参数说明：

    • TAG：标签（版本）；
    • DIGEST：摘要；
    • OPTIONS：
      • -a ：下载镜像仓库中所有的指定镜像；
        – -disable-content-trust：跳过镜像验证（默认值是true）；
        – -platform： 如果服务具有多平台功能，则设置平台；
      • -q：一直详细输出；

    使用 docker pull 命令下载一个指定的镜像或一组镜像（例如，一个存储库），如果没有提供标签，Docker Engine 默认使用:latest标签。这个命令拉出最新的发行版镜像，如：

  • 使用代理

    控制面板 -> 网络 -> 代理服务器

  • 修改 Docker 注册表设置, 启用注册表, 使用国内镜像加速

    注册表 -> 设置 -> 新增 -> 使用

更多加速镜像参考我不是矿神的Docker访问解决方法

群晖Docker 注册表-设置 里面编辑 Docker Hub添加镜像，之后使用ssh连接docker pull 。。。下载相当快。阿里云（香港）之前的几个都不错，推荐微软云！！！不建议阿里云（镜像可能不全！！！！！！）

3.6.2 Docker IPV6

局域网开启了 IPv6 后，发现 Docker 因为没有开启 IPv6 无法访问了，因此需要为 Docker 开启 IPv6，根据官方文档提示，IPv6 仅在运行于 Linux 主机上的 Docker 守护进程上受支持
Docker 支持只给特定的网络开启 IPv6，也支持给 bridge 网络开启 IPv6

最简单的可能就是直接在搭建容器的时候使用host网络，前提是宿主机（也就是 NAS）上已经成功启用了IPv6网络。
弊端就是随着 NAS 上的应用越来越多，端口可能会产生冲突。

根据网上开启 IPV6 的教程，设置完成后，每次群晖重启需要手动更改配置地址（或者使用脚本完成），不够方便优雅。一顿摸索后，发现使用反代最为方便快捷

反向代理服务器间接完成 IPV6 访问：

控制面板 -> 登录门户 -> 高级 -> 反向代理服务器 -> 新增

3.7 WebDAV Server

WebDAV 服务器可让用户编辑和管理存储在远程服务器上的文件。启用 WebDAV 服务器后，支持 WebDAV 的客户端程序 (如某些 Windows 应用程序、Mac OS Finder、Linux 资源营理器) 能够远程访问 Synology NAS，如同访问本地网络硬盘。

  1. 打开 WebDAV Server，并按照图示进行配置
  2. Win10 打开我的电脑，在空白处右键点击 “添加一个网络位置”，之后一直点击下一步，直到填写网络地址处
  3. 按提示填写网址，格式为；
  4. 填写群晖用户名密码（建议勾选记住凭证），点击下一步
  5. 填写网络硬盘名称并点击下一步，之后核对相关信息后点击完成即可完成设置，此时电脑上将会显示所添加的网络硬盘
